# MediaPipe 메이크업 6종 — Stage 1(엔진+UI) 검증 보고서

> **체크포인트용.** 운영자 셀카 확인 전, 엔진이 "기본 베이스 + 6종 포인트"를
> 제대로 만드는지 실제 렌더로 검증. **결론: 작동함.** 12종 모두 얼굴 100%
> 보존 + 베이스로 화사 + 룩별 구분 명확. src/ 이식·Gemini 제거·결제연동은
> **운영자 OK 후 Stage 2**로 보류(현재 main·Gemini 무수정).

## 1. 무엇을 만들었나 (브랜치 `feat/mediapipe-makeup`)
- `looks-config.js` — **단일 소스**: 공통 베이스(여/남) + 12종 룩(색·알파·스타일).
- `looks-engine.js` — 렌더 엔진: 립(tint/matte/gradient/glossy)·블러셔·T존 광채(soft-light)·아이섀도(꺼풀→눈썹 방향, 기울기 대응)·휘도보존 헤어 리컬러. 모자(크라운) 감지 포함.
- `looks.html` — 1개 선택 UI(성별 토글 + 6버튼, "베이스 끄고 비교" 토글). 헤드리스 훅.
- `run-looks.mjs` — 12종 + 베이스 on/off + 모자 케이스 자동 렌더.

## 2. 구조: 기본 베이스 + 룩 포인트
모든 룩에 **항상** 베이스가 깔림 → 칙칙함 해결:
- 입술 자연 혈색(은은한 코랄 틴트) · 볼 자연 블러셔(생기) · T존 하이라이터(광채)
- 각 룩 = 베이스 + 포인트(립/아이/헤어). 립 지정 룩은 베이스 립을 대체.

- **메이크업 룩 처리 ~1.1~1.3s** ✅ (목표 1~2s 충족)
- 헤어 룩 처리: 헤드리스에서 ~10s (아래 한계 참고)

검토 이미지(생성됨):

- `female_LOOKS_montage.png` — 여 6종(글래스/블러드립/란제리/라벤더/K-pop/코퍼)
- `male_LOOKS_montage.png` — 남 6종(글로우/노메이크업/K-팝/스모키/모노크롬/애쉬)
- `eyeshadow_check.png` — 핑크/스모키/테라코타 아이섀도 발색 확인

## 4. 판정
- **얼굴 100% 보존** ✅ — 원본 픽셀 위 합성, 재생성 없음(12종 전부 동일인).
- **화사함(칙칙 해결)** ✅ — 베이스 블러셔+광채가 혈색/생기 부여.
- **룩 구분 명확** ✅ — 립 색/마감(글로시·그라데이션·매트)·아이섀도·헤어로 차별화.
  - 여성: 라벤더 립·블러드 로즈·K-pop 핑크 등 또렷. 블러셔 부드러움(동그란 자국 X).
  - 남성: 의도적으로 은은(립 톤 중심) + 스모키/모노크롬 아이로 구분.
- **1개 선택 작동** ✅ — 버튼 선택 → 그 룩만 즉시(공짜) 재적용.
- **텍스트 버그 없음** ✅ — 오버레이 방식이라 원천적으로 발생 불가.
- **모자/측면** ✅(부분) — 비니 위 메이크업 정상. 3/4 각도 OK. 헤어는 보이는 부분만.

## 5. 한계점 / Stage 2에서 다듬을 것
1. **헤어 룩 성능**: 휘도보존 리컬러가 1024²에서 픽셀 루프(헤드리스 ~10s).
   실기기 GPU에선 훨씬 빠를 전망이나 **운영자 폰에서 헤어 룩 속도 확인 필요**.
   필요 시 다운스케일 리컬러로 최적화(메이크업 룩은 이미 ~1.2s로 빠름).
2. **모자 감지(휴리스틱)**: 총 헤어%로는 짧은 머리(11%)와 비니(11%) 구분 불가 →
   "이마 위 크라운이 머리카락이냐" 방식으로 교체. 현재 비니=0, 짧은머리=0.3,
   긴머리=0.82로 분리(임계 0.15). 완벽하진 않으니 실셀카로 확인 권장.
3. **남성 룩 강도**: 현재 은은. 운영자 취향에 따라 포인트 강도 조정 가능.
4. **다인 사진**: 메이크업은 1명, 헤어 세그는 전원 → 셀카(1인) 전제면 무관.

## 6. 운영자 셀카 확인 방법 (← 지금 이 단계)
```bash
cd experiments/mediapipe-poc
python3 -m http.server 8000      # 또는 npx serve
#  → 폰/PC 브라우저로 http://<코드스페이스>:8000/looks.html
#  → 본인 셀카 업로드 → 성별 선택 → 6개 룩 눌러보며 확인
#  → 특히 본인 모자/측면 셀카로 베이스+포인트 결과 점검
```
헤드리스 재현: `node run-looks.mjs` → `out/`에 결과 생성.
